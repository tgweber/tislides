Mächtigster Berechenbarkeitsbegriff:
Turing-Berechenbarkeit

Turingmaschinen lassen sich kodieren. ($w_M$ steht für den Code für $M$ in einem Wort $w$)

Turingmaschinen können Input für andere Turingmaschinen sein.

Turingmaschinen, die "interessante" Eigenschaften bei anderen Turingmaschinen prüfen.

Das Problem LIAR

Gegeben: eine Turingmaschine $M$, mit Input $w_M$.

Gesucht: 0, wenn Output von $M$ auf $w_M$ 1 ist, 0 sonst.

Annahme: Es gibt eine TM $M_L$, die LIAR erkennt.

Widerspruch!

Als Product Owner:in des Serverless-Computings möchte ich, dass nur terminierende Code-Snippets deployt werden, um die verfügbaren Ressourcen effizient zu nutzen.

Das Problem HALT

Gegeben: eine kodierte Turingmaschine $w_M$ und ein Input $x$.

Gesucht: 1, wenn $M$ mit Input $x$ hält, 0 sonst.

Berechnungstheoeretische Reduktion

$A \leq_B B$

Wenn B berechenbar ist,
dann ist auch A berechenbar.

Kontraposition:
Wenn A nicht berechenbar ist,
dann ist auch B nicht berechenbar.

Das Problem HALT ist nicht berechenbar!
